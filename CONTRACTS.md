# Overlay Cat — module contracts (v1)

Binding interfaces between the three build tracks. Any change to this file requires updating all
dependents. House style: all imports at top; no try/except unless the handler does
something meaningful; dependency injection everywhere; no env vars; relative imports inside
packages; `sys.dont_write_bytecode = True` in entry points only (not in library modules).

## 1. Clip Package (content ↔ runtime)

```
clips/<clip_name>/
  manifest.json
  frames/0000.png ... NNNN.png     # RGBA, premultiplied not required
```

manifest.json fields (all required unless noted):
```json
{
  "name": "sleep_curl",
  "category": "sleep",            // sleep | idle | groom | play | walk | reaction
  "fps": 12.0,
  "frameCount": 108,
  "loop": "pingpong",             // pingpong | forward | once
  "pixelSize": [777, 454],
  "displayScale": 2.0,
  "model": "birefnet_general",    // provenance, optional
  "sourceVideo": "...",           // provenance, optional
  "anchor": [388, 454]            // optional: ground-contact point in px, default bottom-center
}
```

clips/index.json (generated by pipeline/build_index.py):
```json
{ "clips": [ {"name": "sleep_curl", "category": "sleep", "dir": "clips/sleep_curl"} ] }
```

## 2. Brain (overlay/brain/ — pure Python, ZERO AppKit/PyObjC imports)

# overlay/brain/commands.py
@dataclass(frozen=True)
class PlayClip:
    clip: str                 # clip name from index
    loop: str                 # "pingpong" | "forward" | "once"
    min_seconds: float        # selector's chosen duration before next decision
    motion: str | None = None # "left" | "right" on walk commands, None otherwise;
                              # runtime starts locomotion when motion is set and
                              # stops it on the next command with motion=None

@dataclass(frozen=True)
class BrainEvent:             # runtime -> brain
    kind: str                 # "pet" | "click" | "double_click" | "drag_start"
                              # | "drag_end" | "wake"; "double_click" (v1.3) is the
                              # user's deterministic scene shuffle ("show me a
                              # different clip")
    at: float                 # monotonic seconds

# overlay/brain/clock.py
class Clock(Protocol):
    def now(self) -> float: ...          # monotonic seconds
    def wall(self) -> float: ...         # unix epoch seconds (for circadian/persistence)

# overlay/brain/needs.py
class NeedsModel:
    def __init__(self, rng: random.Random, initial: dict[str, float] | None = None): ...
    needs: dict[str, float]  # energy, hunger, playfulness, affection, cleanliness in [0,1]
    def tick(self, dt: float, wall: float) -> None: ...
    def fast_forward(self, elapsed: float) -> None: ...   # clamped at 12h internally
    def apply_event(self, kind: str) -> None: ...
    def set_mode(self, category: str) -> None: ...        # facade sets the active category
                                                          # (tick carries no activity)
    def to_dict(self) -> dict: ...
    @classmethod
    def from_dict(cls, d: dict, rng: random.Random) -> "NeedsModel": ...

# overlay/brain/selector.py
class ActivitySelector:
    def __init__(self, catalog: list[Activity], rng: random.Random,
                 temperature: float = 0.6): ...
    def choose(self, needs: dict[str, float], wall: float) -> Activity: ...
    # softmax over utility; recency penalty w/ exponential recovery; hysteresis for sleep
    # BREAKING (v1.3): the v1.2 exclude= parameter is gone — double-click is now
    # a clip-level scene shuffle owned by the facade (see below), kept in sync
    # via force()
    def force(self, activity: Activity, wall: float) -> None: ...
    # facade shuffle path: adopt the activity as current with the exact
    # deselection bookkeeping (recency stamp + cooldown on the one left) that
    # choose() applies on a switch; forcing the current activity again is a
    # no-op (its min-duration window keeps its original start); any pending
    # release() is cleared — the forced activity starts fully sticky
    def release(self) -> None: ...        # facade-approved preemption: lifts min-duration
                                          # stickiness for the next choose() only
    def wake(self, wall: float) -> None:  # click-wake: exit sleep + short re-entry cooldown
        ...

# overlay/brain/activities.py
@dataclass(frozen=True)
class Activity:
    name: str                  # "sleep", "idle", "groom"...
    clips: tuple[str, ...]     # candidate clip names (selector/runtime picks one via rng)
    category: str
    utility_need: str          # which need drives it (e.g. sleep <- 1-energy)
    invert: bool               # True if utility = need value, False if 1-need
    min_s: float
    max_s: float               # duration sampled log-normal-ish between min/max
    motions: tuple[str | None, ...]  # parallel to clips; "left"/"right" on walk
                                     # clips (derived from the name), None otherwise

@dataclass(frozen=True)
class Catalog:                 # build_catalog output; the facade's full clip knowledge
    activities: tuple[Activity, ...]   # schedulable activities (never "reaction")
    durations: dict[str, float]        # EVERY indexed clip -> frameCount / fps seconds
    reaction_clips: tuple[str, ...]    # reaction-category clips (facade-only)

def walk_direction(name: str) -> str | None:
    # "left" if "walk_left" in name, "right" if "walk_right"; None otherwise;
    # raises ValueError if a name contains both substrings
def build_catalog(index: dict, manifests: dict[str, dict]) -> Catalog:
    # BREAKING (v1.1): manifests (clip name -> manifest.json content, as the
    # orchestrator already loads) is now required for durations. Raises ValueError
    # on unknown category, missing manifest, non-positive fps/frameCount, or a
    # walk-category clip whose name lacks a direction substring (loud, no fallback).

# overlay/brain/facade.py
class Brain:
    def __init__(self, clock: Clock, rng: random.Random, catalog: Catalog,
                 store: PersistenceStore): ...
    def tick(self, events: list[BrainEvent]) -> PlayClip | None:
        # called at 1 Hz + immediately after events; returns a command ONLY when
        # the activity changes (runtime keeps playing current clip otherwise)
    def shutdown(self) -> None: ...      # persist state

# Facade behavior addenda (v1.1):
# - Walk: when the walk activity is chosen the facade picks a direction at random,
#   then a clip of that direction, and emits PlayClip(..., loop="forward",
#   motion="left"|"right"). All other commands carry motion=None.
# - Reactions: on a "click" event while NOT sleeping (and no "double_click" in the
#   same batch), if reaction_clips is non-empty and >= 10 s passed since the last
#   reaction (REACTION_COOLDOWN_S), tick() emits
#   PlayClip(reaction, "once", min_seconds=durations[reaction], motion=None). Until
#   that duration elapses tick() emits nothing (events still feed needs); at the
#   first tick after completion it re-emits the interrupted activity's EXACT clip
#   (motion preserved for walk) so the cat resumes in place — no teleporting.
#   Clicks during the cooldown keep the pre-existing behavior (runtime boop).
#   Click on a sleeping cat stays a wake, never a reaction.

# Deterministic event semantics (v1.2 — replaces the old 5 s event-preempt gate;
# EVENT_PREEMPT_MIN_S is gone):
# - "pet" / "drag_start" / "drag_end": update needs ONLY. Never reconsider, never
#   release. A carried or petted cat resumes exactly what it was doing.
# - "click": reaction path above; during the 10 s cooldown the brain emits NOTHING
#   (runtime boops). Click on a sleeping cat wakes it.
# - "double_click" (v1.3 — SCENE SHUFFLE; replaces the v1.2 activity re-roll):
#   pick UNIFORMLY at random over every clip of every schedulable activity
#   (the full Catalog.activities pool — reaction clips are never in it; a walk
#   clip carries its direction as motion), EXCLUDING the currently playing
#   clip. Sleep's hysteresis gate and ALL cooldowns are bypassed: this is an
#   explicit user command. The pick is emitted with its activity's
#   LOOP_BY_CATEGORY mode and a fresh duration sampled from that activity's
#   min_s/max_s; internal state (selector current via force(), needs mode,
#   deadline) switches exactly as if selected normally. Guarantee: with >= 2
#   clips in the pool the scene ALWAYS visibly changes (different clip, even
#   when the activity repeats); a single-clip catalog emits nothing. While
#   sleeping the shuffle still runs — waking implicitly, with selector.wake()
#   bookkeeping so hysteresis cannot insta-return to sleep (a sleep clip may
#   still be the explicit pick). During a reaction interlude it cancels the
#   interlude and shuffles (single-clip catalogs resume the prior clip in
#   place instead).
# - "wake": needs fast-forward over the wall-clock gap + reconsider with release()
#   (unchanged).

# overlay/brain/store.py
class PersistenceStore:                  # dumb layer: stores exactly what it receives
    def __init__(self, path: Path): ...
    def load(self) -> dict | None: ...
    def save(self, data: dict) -> None: ...

## 3. Runtime (overlay/app/ — PyObjC; owns ALL AppKit)

```python
# overlay/app/renderer.py
class Renderer(Protocol):
    def available_clips(self) -> dict[str, dict]: ...        # name -> manifest
    def play(self, clip: str, loop: str) -> None: ...        # crossfades 200ms between clips
                                                             # of near-equal point size; when
                                                             # the two clips' point sizes differ
                                                             # >15% in either dimension the
                                                             # switch hard-cuts instead
                                                             # (outgoing opacity 0 immediately —
                                                             # a stretched outgoing pose reads
                                                             # as a deformed second cat)
    def current_clip(self) -> str | None: ...                # name now playing (edge swap
                                                             # reads it to derive the pair)
    def current_alpha_at(self, x_pt: float, y_pt_bottom: float) -> int: ...
    def window_frame(self) -> tuple[float, float, float, float]: ...
    def set_frame_origin(self, x: float, y: float) -> None: ...
    def anchor_offset(self) -> tuple[float, float]: ...      # current clip's manifest anchor
                                                             # as points from window bottom-left
    def boop(self) -> None: ...                              # click reaction animation

# SpriteRenderer: CALayer-based implementation (two layers for crossfade), per-clip
# NSTimer at native fps, ping-pong/forward/once; frames are never kept decoded —
# each tick creates the displayed frame's NSImage from disk and the CALayer's
# retention bounds decode-cache memory to the on-screen frame(s). Per-clip RAM is
# frame paths + np.packbits alpha masks (threshold 24), current clip only (LRU=1);
# the previous clip survives only as the outgoing layer's retained last frame.

# User scale (v1.3 — continuous slider replaces the Size presets): the
# status-menu slider sets any scale in [0.10, 10.0] (10%..1000%) through a
# LOGARITHMIC mapping (status_item.slider_to_scale: v in 0..1 ->
# 0.10 * (10.0/0.10)**v, so 100% sits mid-slider); the action fires
# continuously during drag and the orchestrator applies it live.
# SCREEN-FIT CLAMP: OverlayWindow.resize_to_clip clamps the EFFECTIVE scale
# (window.clamp_user_scale, SCREEN_FIT_FRACTION = 0.95) so the scaled clip
# never exceeds 95% of NSScreen.mainScreen().visibleFrame in either dimension
# (one factor, aspect kept). The window is the single source of truth for the
# effective scale: effective_scale(manifest) = displayScale /
# effective_user_scale EVERYWHERE points meet pixels — resize_to_clip sizing,
# anchor_offset, current_alpha_at, and therefore Locomotion.floor_window_y.
# user_scale() keeps returning the REQUESTED scale (what persists and what the
# label shows); scale_clamped() reports requested != effective for the current
# clip (False before any clip). OverlayWindow.set_user_scale(v) re-applies the
# current clip's geometry immediately with the anchor point fixed on screen.
# The crossfade size gate (renderer.crossfade_allowed) compares manifest point
# sizes only — user_scale multiplies both clips identically and never affects
# the fade decision.

# overlay/app/settings.py
class SettingsStore:        # dumb JSON layer: stores exactly what it receives
    def __init__(self, path: Path): ...
    def load(self) -> dict | None: ...   # None when the file does not exist
    def save(self, data: dict) -> None: ...
# Settings shape (owned by the ORCHESTRATOR, the store never inspects it):
#   {"user_scale": float > 0}   # REQUESTED scale, e.g. 2.5 = 250%; the old
#                               # preset values are valid floats — no migration
# Default path ~/Library/Application Support/OverlayCat/settings.json; the
# orchestrator loads it at start (missing file -> 1.0; present-but-invalid
# user_scale raises — no silent fallback) and saves on every slider action
# (continuous during drag; the file is tiny JSON).

# overlay/app/interactions.py
class InteractionMonitor:   # polling + event translation; emits BrainEvent via callback
    def __init__(self, renderer: Renderer, on_event: Callable[[BrainEvent], None]): ...
    def is_dragging(self) -> bool: ...   # True from first dragged movement until mouse-up
    # 1 Hz proximity check escalating to 15 Hz mouseLocation polling near the window;
    # toggles panel.ignoresMouseEvents from alpha sampling; stroke detection for "pet"
    # (>=120 px path length over cat pixels within 0.7 s rolling window)
    # Click vs double-click (v1.2): a non-drag mouse-up with clickCount 1 schedules
    # the "click" emission on a 0.25 s one-shot NSTimer (CLICK_DEBOUNCE_S); a
    # mouse-up with clickCount 2 cancels that timer and emits "double_click" — a
    # double-click NEVER also emits "click". The boop animation stays immediate on
    # every non-drag mouse-up. pause() cancels any pending click emission.

# overlay/app/locomotion.py
WALK_SPEED_PT_S = 60.0      # default; a walk clip manifest may override with an
                            # optional "walkSpeedPtS" number (orchestrator reads it)
class Locomotion:           # owns walk velocity, floor line, screen-edge detection
    def __init__(self, renderer: Renderer, on_edge_swap: Callable[[str], None],
                 drag_active: Callable[[], bool]): ...
    def start(self, direction: str, speed_pt_s: float) -> None: ...  # "left" | "right"
    def stop(self) -> None: ...          # orchestrator calls on any non-motion PlayClip
    def pause(self) -> None: ...         # app-level pause; walk state survives
    def resume(self) -> None: ...
    def active(self) -> bool: ...
    def direction(self) -> str | None: ...
    def floor_window_y(self) -> float: ...   # visibleFrame.origin.y - anchor_offset()[1]
    # Own 30 Hz NSTimer (invalidated whenever not walking/paused) moves the window
    # horizontally with renderer.set_frame_origin, pinning y so the clip anchor rides
    # NSScreen.visibleFrame's bottom. When the leading window edge comes within 8 pt
    # of a visibleFrame side it reverses velocity and fires on_edge_swap(new_direction);
    # the ORCHESTRATOR then swaps to the opposite clip of the pair by replacing the
    # 'walk_left'/'walk_right' name substring and calling renderer.play — no brain
    # round trip (raises if the name has no substring or the opposite clip is absent).
    # While drag_active() is True ticks are no-ops; movement resumes from the drop x.

# overlay/app/orchestrator.py
class AppOrchestrator:       # composition root; the ONLY place objects are wired
    def __init__(self, clips_index_path: Path, state_path: Path,
                 settings_path: Path | None = None,
                 brain_factory: Callable | None = None): ...
    # brain_factory(index, state_path, clock, rng) -> Brain-shaped object is an
    # optional test/smoke seam; production (None) wires the real overlay.brain.
    # settings_path None means DEFAULT_SETTINGS_PATH
    # (~/Library/Application Support/OverlayCat/settings.json).
    def run(self) -> None: ...
    def set_user_scale(self, value: float) -> bool: ...
    # Size-slider handler (fires continuously during drag): window.set_user_scale
    # (geometry re-applied immediately, anchor fixed) then
    # SettingsStore.save({"user_scale": value}); returns window.scale_clamped()
    # so the slider label can append "(max)". After every renderer.play the
    # orchestrator pushes window.scale_clamped() to
    # StatusItemController.set_clamped (clips differ in size, so the clamp
    # state can change without a slider move).
    # Command handling: every PlayClip goes to renderer.play; PlayClip.motion
    # (contract-optional, default None — absent attribute on Brain-shaped seam
    # objects means None) starts Locomotion with the manifest-aware walk speed,
    # and any non-motion command stops it. The 1 Hz brain tick timer keeps
    # running while a loop="once" clip holds its last frame, so the brain's
    # post-reaction resume command arrives on schedule and crossfades back.

# overlay/app/status_item.py — 🐱 menu: Pause/Resume; a "Size" header plus an
# NSMenuItem hosting a custom NSView via setView_ (public AppKit) with a
# CONTINUOUS 220 pt NSSlider (range 0..1, log-mapped to 10%..1000%) and a live
# NSTextField label showing the REQUESTED percent, "(max)" suffixed while the
# window clamps; Quit. Pure helpers slider_to_scale / scale_to_slider /
# format_scale_label are module-level and AppKit-free for tests.

# overlay/app/main.py — entry point:
#   python -B -m overlay.app.main --clips clips [--state ~/Library/.../overlaycat.json]
#       [--settings ~/Library/.../settings.json]
```

Threading: everything on the main thread via NSTimer on NSRunLoopCommonModes (constants, not
literal strings). Brain tick timer: 1 Hz with 0.3 s tolerance. Locomotion timer: 30 Hz, alive
only while walking. All timers invalidated when NSWindow occlusion reports hidden or screen
sleeps; needs fast-forward on resume.

Live verification: tests/integration/verify_live.py (frames/hit-toggle/crossfade/capture) and
tests/integration/verify_walk.py (locomotion speed + floor line + edge swap against clips_3d).

## 4. Working PyObjC reference

`overlay/spike_overlay.py` is the proven-on-this-machine reference for: panel style mask &
collection behavior, CALayer contents swap, alpha hit-testing math (displayScale 2.0, top-left
pixel origin vs bottom-left window origin), drag, status item, timer scheduling. Reuse its
patterns; it stays untouched as a fallback demo.

## 5. Known machine constraints

torch==2.8.0 pinned (2.9+ drops MPS on macOS 13). SwiftPM broken under CLT here — Python only.
Never write the literal string "NSRunLoopCommonModes" — import the constant. Never rely on
automatic per-pixel click-through; we own ignoresMouseEvents from first toggle.
